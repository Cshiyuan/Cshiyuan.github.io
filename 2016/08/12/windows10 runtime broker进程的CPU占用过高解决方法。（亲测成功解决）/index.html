windows10 runtime broker进程的CPU占用过高解决方法。（亲测成功解决）

  1. 关闭onedrive，不赘言
  2. 关闭自动更新，选择提醒和延迟
  3. 锁屏界面不用windows聚焦——关于这点，我直接选择了某张图片
  4. 设置->系统->通知和操作->禁用"显示有关windows的提示"——关于这点，我把所有的通知都关闭了
  5. 开始菜单-照片-设置，关闭选项——关于这点，我是把所有选项都关闭
  6. 运行 regedit.exe, 逐次打开HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\TimeBroker,改变"Start"的值:00000003 为 值:00000004.
  7. 放一张全新的图片到我的电脑-图片里，然后把这张图片设置成桌面。——关于这点，我怀疑我就是昨晚这个操作之后，进程才回复正常的，前半部分操作是有人提到的，后半部分操作是我心血来潮的，有兴趣大家可以试试，只放图片，不改桌面。不过我个人心里也是担心，使用windows的主题图片群是否对此有影响。

 

上面是我找到的有关方法，但是其实最重要的是！！

设置->系统->通知和操作->禁用"显示有关windows的提示"

这点是最TM重要的了。
