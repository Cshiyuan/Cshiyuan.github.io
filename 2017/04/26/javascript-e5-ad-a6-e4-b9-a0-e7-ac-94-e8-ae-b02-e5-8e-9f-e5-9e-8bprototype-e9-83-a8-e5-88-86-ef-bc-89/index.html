JavaScript学习笔记(2)(原型prototype部分）

原型是指当我们想要创建xiaoming这个具体的学生时，我们并没有一个Student类型可用。那怎么办？恰好有这么一个现成的对象：

<span class="keyword">var</span> robot = {
    name: <span class="string">'Robot'</span>,
    height: <span class="number">1.6</span>,
    run: <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
        console.log(<span class="keyword">this</span>.name + <span class="string">' is running...'</span>);
    }
};
`</pre>
我们看这个`robot`对象有名字，有身高，还会跑，有点像小明，干脆就根据它来“创建”小明得了！

于是我们把它改名为`Student`，然后创建出`xiaoming`：
<pre>`<span class="keyword">var</span> Student = {
    name: <span class="string">'Robot'</span>,
    height: <span class="number">1.2</span>,
    run: <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
        console.log(<span class="keyword">this</span>.name + <span class="string">' is running...'</span>);
    }
};

<span class="keyword">var</span> xiaoming = {
    name: <span class="string">'小明'</span>
};

xiaoming.__proto__ = Student;
`</pre>
注意最后一行代码把`xiaoming`的原型指向了对象`Student`，看上去`xiaoming`仿佛是从`Student`继承下来的：
<pre>`xiaoming.name; <span class="regexp">//</span> <span class="string">'小明'</span>
xiaoming.run(); <span class="regexp">//</span> 小明 is running...
`</pre>
`xiaoming`有自己的`name`属性，但并没有定义`run()`方法。不过，由于小明是从`Student`继承而来，只要`Student`有`run()`方法，`xiaoming`也可以调用：

![xiaoming-prototype](http://www.liaoxuefeng.com/files/attachments/001435287613668a73ab76ccc85411282c1b1370be41636000/l)

JavaScript的原型链和Java的Class区别就在，它没有“Class”的概念，所有对象都是实例，所谓继承关系不过是把一个对象的原型指向另一个对象而已。

如果你把`xiaoming`的原型指向其他对象：
<pre>`<span class="keyword">var</span> Bird = {
    fly: <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
        console.log(<span class="keyword">this</span>.name + <span class="string">' is flying...'</span>);
    }
};

xiaoming.__proto__ = Bird;
`</pre>
现在`xiaoming`已经无法`run()`了，他已经变成了一只鸟：
<pre>`xiaoming.fly(); <span class="regexp">//</span> 小明 is flying...
`</pre>
在JavaScrip代码运行时期，你可以把`xiaoming`从`Student`变成`Bird`，或者变成任何对象。

_请注意_，上述代码仅用于演示目的。在编写JavaScript代码时，不要直接用`obj.__proto__`去改变一个对象的原型，并且，低版本的IE也无法使用`__proto__`。`Object.create()`方法可以传入一个原型对象，并创建一个基于该原型的新对象，但是新对象什么属性都没有，因此，我们可以编写一个函数来创建`xiaoming`：
<pre>`<span class="comment">// 原型对象:</span>
<span class="keyword">var</span> Student = {
    name: <span class="string">'Robot'</span>,
    height: <span class="number">1.2</span>,
    run: <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
        console.log(<span class="keyword">this</span>.name + <span class="string">' is running...'</span>);
    }
};

<span class="function"><span class="keyword">function</span> <span class="title">createStudent</span><span class="params">(name)</span> {</span>
    <span class="comment">// 基于Student原型创建一个新对象:</span>
    <span class="keyword">var</span> s = Object.create(Student);
    <span class="comment">// 初始化新对象:</span>
    s.name = name;
    <span class="keyword">return</span> s;
}

<span class="keyword">var</span> xiaoming = createStudent(<span class="string">'小明'</span>);
xiaoming.run(); <span class="comment">// 小明 is running...</span>
xiaoming.__proto__ === Student; <span class="comment">// true</span>
`</pre>
这是创建原型继承的一种方法，JavaScript还有其他方法来创建对象，我们在后面会一一讲到。

## 创建对象

* * *

<div class="x-wiki-content">

JavaScript对每个创建的对象都会设置一个原型，指向它的原型对象。

当我们用`obj.xxx`访问一个对象的属性时，JavaScript引擎先在当前对象上查找该属性，如果没有找到，就到其原型对象上找，如果还没有找到，就一直上溯到`Object.prototype`对象，最后，如果还没有找到，就只能返回`undefined`。

例如，创建一个`Array`对象：
<pre>`<span class="keyword">var</span> arr = [<span class="number">1</span>, <span class="number">2</span>, <span class="number">3</span>];
`</pre>
其原型链是：
<pre>`arr ----&gt; <span class="keyword">Array</span>.prototype ----&gt; Object.prototype ----&gt; <span class="keyword">null</span>
`</pre>
`Array.prototype`定义了`indexOf()`、`shift()`等方法，因此你可以在所有的`Array`对象上直接调用这些方法。

* * *

</div>

## 构造函数

除了直接用`{ ... }`创建一个对象外，JavaScript还可以用一种构造函数的方法来创建对象。它的用法是，先定义一个构造函数：
<pre>`<span class="function"><span class="keyword">function</span> <span class="title">Student</span><span class="params">(name)</span> {</span>
    <span class="keyword">this</span>.name = name;
    <span class="keyword">this</span>.hello = <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
        alert(<span class="string">'Hello, '</span> + <span class="keyword">this</span>.name + <span class="string">'!'</span>);
    }
}
`</pre>
你会问，咦，这不是一个普通函数吗？

这确实是一个普通函数，但是在JavaScript中，可以用关键字`new`来调用这个函数，并返回一个对象：
<pre>`<span class="keyword">var</span> xiaoming = <span class="keyword">new</span> Student(<span class="string">'小明'</span>);
xiaoming.name; <span class="comment">// '小明'</span>
xiaoming.hello(); <span class="comment">// Hello, 小明!</span>
`</pre>
_注意_，如果不写`new`，这就是一个普通函数，它返回`undefined`。但是，如果写了`new`，它就变成了一个构造函数，它绑定的`this`指向新创建的对象，并默认返回`this`，也就是说，不需要在最后写`return this;`。

新创建的`xiaoming`的原型链是：
<pre>`xiaoming ----&gt; Student.prototype ----&gt; Object.prototype ----&gt; <span class="literal">null</span>
`</pre>
也就是说，`xiaoming`的原型指向函数`Student`的原型。如果你又创建了`xiaohong`、`xiaojun`，那么这些对象的原型与`xiaoming`是一样的：
<pre>`xiaoming ↘
xiaohong -→ Student.prototype ----&gt; Object.prototype ----&gt; <span class="literal">null</span>
xiaojun  ↗
`</pre>
用`new Student()`创建的对象还从原型上获得了一个`constructor`属性，它指向函数`Student`本身：
<pre>`xiaoming.constructor === <span class="constant">Student</span>.prototype.constructor; <span class="regexp">//</span> <span class="keyword">true</span>
<span class="constant">Student</span>.prototype.constructor === <span class="constant">Student</span>; <span class="regexp">//</span> <span class="keyword">true</span>

<span class="constant">Object</span>.getPrototypeOf(xiaoming) === <span class="constant">Student</span>.prototype; <span class="regexp">//</span> <span class="keyword">true</span>

xiaoming instanceof <span class="constant">Student</span>; <span class="regexp">//</span> <span class="keyword">true</span>
`</pre>
看晕了吧？用一张图来表示这些乱七八糟的关系就是：

![protos](http://www.liaoxuefeng.com/files/attachments/00143529922671163eebb527bc14547ac11363bf186557d000/l)

红色箭头是原型链。注意，`Student.prototype`指向的对象就是`xiaoming`、`xiaohong`的原型对象，这个原型对象自己还有个属性`constructor`，指向`Student`函数本身。

另外，函数`Student`恰好有个属性`prototype`指向`xiaoming`、`xiaohong`的原型对象，但是`xiaoming`、`xiaohong`这些对象可没有`prototype`这个属性，不过可以用`__proto__`这个非标准用法来查看。

现在我们就认为`xiaoming`、`xiaohong`这些对象“继承”自`Student`。

不过还有一个小问题，注意观察：
<pre>`xiaoming.name; <span class="regexp">//</span> <span class="string">'小明'</span>
xiaohong.name; <span class="regexp">//</span> <span class="string">'小红'</span>
xiaoming.hello; <span class="regexp">//</span> <span class="symbol">function:</span> <span class="constant">Student</span>.hello()
xiaohong.hello; <span class="regexp">//</span> <span class="symbol">function:</span> <span class="constant">Student</span>.hello()
xiaoming.hello === xiaohong.hello; <span class="regexp">//</span> <span class="keyword">false</span>
`</pre>
`xiaoming`和`xiaohong`各自的`name`不同，这是对的，否则我们无法区分谁是谁了。

`xiaoming`和`xiaohong`各自的`hello`是一个函数，但它们是两个不同的函数，虽然函数名称和代码都是相同的！

如果我们通过`new Student()`创建了很多对象，这些对象的`hello`函数实际上只需要共享同一个函数就可以了，这样可以节省很多内存。

要让创建的对象共享一个`hello`函数，根据对象的属性查找原则，我们只要把`hello`函数移动到`xiaoming`、`xiaohong`这些对象共同的原型上就可以了，也就是`Student.prototype`：

![protos2](http://www.liaoxuefeng.com/files/attachments/001435299854512faf32868f60348be878982909b5a5d04000/l)

修改代码如下：
<pre>`<span class="function"><span class="keyword">function</span> <span class="title">Student</span><span class="params">(name)</span> {</span>
    <span class="keyword">this</span>.name = name;
}

Student.prototype.hello = <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
    alert(<span class="string">'Hello, '</span> + <span class="keyword">this</span>.name + <span class="string">'!'</span>);
};
`</pre>
用`new`创建基于原型的JavaScript的对象就是这么简单！

## 忘记写new怎么办

如果一个函数被定义为用于创建对象的构造函数，但是调用时忘记了写`new`怎么办？

在strict模式下，`this.name = name`将报错，因为`this`绑定为`undefined`，在非strict模式下，`this.name = name`不报错，因为`this`绑定为`window`，于是无意间创建了全局变量`name`，并且返回`undefined`，这个结果更糟糕。

所以，调用构造函数千万不要忘记写`new`。为了区分普通函数和构造函数，按照约定，构造函数首字母应当大写，而普通函数首字母应当小写，这样，一些语法检查工具如[jslint](http://www.jslint.com/)将可以帮你检测到漏写的`new`。

最后，我们还可以编写一个`createStudent()`函数，在内部封装所有的`new`操作。一个常用的编程模式像这样：
<pre>`<span class="function"><span class="keyword">function</span> <span class="title">Student</span><span class="params">(props)</span> {</span>
    <span class="keyword">this</span>.name = props.name || <span class="string">'匿名'</span>; <span class="comment">// 默认值为'匿名'</span>
    <span class="keyword">this</span>.grade = props.grade || <span class="number">1</span>; <span class="comment">// 默认值为1</span>
}

Student.prototype.hello = <span class="function"><span class="keyword">function</span> <span class="params">()</span> {</span>
    alert(<span class="string">'Hello, '</span> + <span class="keyword">this</span>.name + <span class="string">'!'</span>);
};

<span class="function"><span class="keyword">function</span> <span class="title">createStudent</span><span class="params">(props)</span> {</span>
    <span class="keyword">return</span> <span class="keyword">new</span> Student(props || {})
}
`</pre>
这个`createStudent()`函数有几个巨大的优点：一是不需要`new`来调用，二是参数非常灵活，可以不传，也可以这么传：
<pre>`<span class="keyword">var</span> xiaoming = createStudent({
    name: <span class="string">'小明'</span>
});

xiaoming.grade; <span class="comment">// 1</span>

如果创建的对象有很多属性，我们只需要传递需要的某些属性，剩下的属性可以用默认值。由于参数是一个Object，我们无需记忆参数的顺序。如果恰好从JSON拿到了一个对象，就可以直接创建出xiaoming。
